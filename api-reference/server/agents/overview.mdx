Pipecat is a multi-agent framework that lets you build AI applications with multiple coordinated agents. Agents communicate through a shared bus, can activate and deactivate each other, and coordinate work through structured job protocols.

## Installation

The multi-agent framework ships with `pipecat-ai`:

```bash
uv add pipecat-ai
```

Optional extras for specific features:

```bash
# Redis distributed bus
uv add "pipecat-ai[redis]"

# PGMQ distributed bus
uv add "pipecat-ai[pgmq]"

# WebSocket proxy agents
uv add "pipecat-ai[websocket]"
```

## Agent Type Hierarchy

| Class                                                                             | Parent           | Description                                          |
| --------------------------------------------------------------------------------- | ---------------- | ---------------------------------------------------- |
| [`BaseWorker`](/api-reference/server/agents/base-agent)                                  | -                | Core agent with lifecycle, bus, and job coordination |
| [`PipelineWorker`](/api-reference/server/agents/base-agent)                              | `BaseWorker`     | Adds a Pipecat pipeline runtime                      |
| [`LLMWorker`](/api-reference/server/agents/llm-agent)                                    | `PipelineWorker` | Adds LLM pipeline and `@tool` registration           |

| [`WebSocketProxyServer`](/api-reference/server/agents/proxy-agents#websocketproxyserver) | `BaseWorker`     | Receives bus messages from a remote client           |
