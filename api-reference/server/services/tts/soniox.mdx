## Overview

`SonioxTTSService` provides real-time text-to-speech synthesis using Soniox's WebSocket API with support for over 60 languages, multiple voices, and streaming audio generation optimized for conversational AI use cases.

## Installation

To use Soniox TTS services, install the required dependencies:

```bash
pip install "pipecat-ai[soniox]"
```

## Prerequisites

### Soniox Account Setup

Before using Soniox TTS services, you need:

1. **Soniox Account**: Sign up at [Soniox Console](https://console.soniox.com/)
2. **API Key**: Generate an API key from your console dashboard
3. **Voice Selection**: Choose from available [voices](https://soniox.com/docs/tts/concepts/voices) and [models](https://soniox.com/docs/tts/models)

### Required Environment Variables

- `SONIOX_API_KEY`: Your Soniox API key for authentication

## Configuration

### SonioxTTSService

<ParamField path="api_key" type="str" required>
  Soniox API key for authentication.
</ParamField>

<ParamField path="url" type="str" default="wss://tts-rt.soniox.com/tts-websocket">
  Soniox TTS WebSocket API URL.
</ParamField>

<ParamField path="sample_rate" type="int" default="None">
  Audio sample rate in Hz. When `None`, uses 24kHz.
</ParamField>

<ParamField path="encoding" type="str" default="pcm_s16le">
  Audio encoding format. See [Soniox audio
  formats](https://soniox.com/docs/tts/concepts/audio-formats) for
  available options.
</ParamField>

<ParamField path="settings" type="SonioxTTSService.Settings" default="None">
  Runtime-configurable settings for the TTS service. See [Settings](#settings)
  below.
</ParamField>

<ParamField
  path="text_aggregation_mode"
  type="TextAggregationMode"
  default="None"
>
  How to aggregate incoming text before synthesis. See [Text
  Aggregation](/guides/learn/text-to-speech#text-aggregation) for details.
</ParamField>

### Settings

Runtime-configurable settings passed via the `settings` constructor argument using `SonioxTTSService.Settings(...)`. These can be updated mid-conversation with `TTSUpdateSettingsFrame`. See [Service Settings](/guides/fundamentals/service-settings) for details.

| Parameter  | Type              | Default               | Description                                                       |
| ---------- | ----------------- | --------------------- | ----------------------------------------------------------------- |
| `model`    | `str`             | `"tts-rt-v1-preview"` | Model to use for synthesis. _(Inherited from base TTS settings.)_ |
| `voice`    | `str`             | `"Maya"`            | Voice identifier. _(Inherited from base TTS settings.)_           |
| `language` | `Language \| str` | `Language.EN`         | Language for synthesis. _(Inherited from base TTS settings.)_     |
| `bitrate`  | `int`             | `None`                | Codec bitrate in bps (for compressed formats).                    |

## Usage

### Basic Setup

```python
from pipecat.services.soniox.tts import SonioxTTSService
from pipecat.transcriptions.language import Language

tts = SonioxTTSService(
    api_key=os.getenv("SONIOX_API_KEY"),
    settings=SonioxTTSService.Settings(
        voice="Adrian",
        language=Language.EN,
    ),
)
```

### With Custom Sample Rate and Encoding

```python
from pipecat.services.soniox.tts import SonioxTTSService

tts = SonioxTTSService(
    api_key=os.getenv("SONIOX_API_KEY"),
    sample_rate=16000,
    encoding="pcm_s16le",
    settings=SonioxTTSService.Settings(
        voice="Adrian",
    ),
)
```

## Notes

- **WebSocket-based service**: Real-time streaming with interruption handling for interactive conversations
- **Partial text input**: Supports streaming text for real-time synthesis
- **Language support**: 60+ languages. See [Soniox TTS languages](https://soniox.com/docs/tts/concepts/languages)
- **Audio formats**: PCM, mulaw, alaw. See [Soniox audio formats](https://soniox.com/docs/tts/concepts/audio-formats)

## Event Handlers

Soniox TTS supports the standard [service connection events](/api-reference/server/events/service-events):

| Event                 | Description                            |
| --------------------- | -------------------------------------- |
| `on_connected`        | Connected to Soniox TTS WebSocket      |
| `on_disconnected`     | Disconnected from Soniox TTS WebSocket |
| `on_connection_error` | Connection error occurred              |

```python
@tts.event_handler("on_connected")
async def on_connected(service):
    print("Connected to Soniox TTS")
```
