---
title: "Tasks"
description: "Dispatch parallel work and coordinate results with task groups in Pipecat Subagents."
---

Tasks let a parent agent dispatch work to one or more workers, wait for results, and optionally cancel or timeout. Workers can send a single response, stream incremental results, or provide progress updates.

## Task Lifecycle

```
              Parent                                Worker
                 │                                    │
                 │  request_task(payload, timeout)     │
                 ├───────────────────────────────────►│  on_task_request()
                 │                                    │
                 │           send_task_update()        │
  on_task_update │◄───────────────────────────────────┤  (optional)
                 │                                    │
                 │     send_task_stream_start/data/end │
  on_task_stream │◄───────────────────────────────────┤  (optional)
                 │                                    │
                 │        send_task_response(status)   │
  on_task_response│◄──────────────────────────────────┤
  on_task_completed│                                  │
```

## Worker: Handling Tasks

### Basic Task Handler

Use the `@task` decorator to handle incoming task requests:

```python
from pipecat_subagents.agents import BaseAgent
from pipecat_subagents.agents.task_decorator import task


class ResearchWorker(BaseAgent):
    @task
    async def on_task_request(self, message):
        topic = message.payload["topic"]
        result = await do_research(topic)
        await self.send_task_response({"findings": result})
```

### Named Tasks

Route different request types to different handlers:

```python
class AnalysisWorker(BaseAgent):
    @task(name="summarize")
    async def handle_summarize(self, message):
        text = message.payload["text"]
        summary = await summarize(text)
        await self.send_task_response({"summary": summary})

    @task(name="sentiment")
    async def handle_sentiment(self, message):
        text = message.payload["text"]
        sentiment = await analyze_sentiment(text)
        await self.send_task_response({"sentiment": sentiment})
```

### Parallel Task Execution

By default, task handlers run sequentially. Use `parallel=True` to handle multiple requests concurrently:

```python
class Worker(BaseAgent):
    @task(parallel=True)
    async def on_task_request(self, message):
        # Each request runs in a separate asyncio task
        result = await long_running_work(message.payload)
        await self.send_task_response(result)
```

### Progress Updates

Send progress updates before the final response:

```python
@task
async def on_task_request(self, message):
    await self.send_task_update({"progress": 0, "status": "starting"})

    partial = await step_one(message.payload)
    await self.send_task_update({"progress": 50, "status": "halfway"})

    result = await step_two(partial)
    await self.send_task_update({"progress": 100, "status": "complete"})

    await self.send_task_response({"result": result})
```

### Streaming Results

Stream incremental results for long-running tasks:

```python
@task
async def on_task_request(self, message):
    await self.send_task_stream_start()

    async for chunk in process_in_chunks(message.payload):
        await self.send_task_stream_data({"chunk": chunk})

    await self.send_task_stream_end()
    await self.send_task_response({"status": "done"})
```

### Task Status

Workers respond with a status:

| Status      | Meaning                                     |
| ----------- | ------------------------------------------- |
| `COMPLETED` | Task finished successfully (default)        |
| `FAILED`    | Task failed due to a logical/business error |
| `ERROR`     | Unexpected runtime error                    |
| `CANCELLED` | Task was cancelled by the requester         |

```python
from pipecat_subagents.agents.task_context import TaskStatus

await self.send_task_response(
    {"error": "Invalid input"},
    status=TaskStatus.FAILED
)
```

## Parent: Dispatching Tasks

### Fire-and-Forget

Send a task without waiting for results:

```python
await self.request_task(
    "worker1",
    payload={"topic": "AI safety"},
    name="research",       # Optional: routes to named handler
    timeout=30.0,          # Optional: timeout in seconds
)
```

Results arrive via `on_task_response()` and `on_task_completed()` callbacks.

### Task Groups

For structured parallel dispatch, use `task_group()`:

```python
async with self.task_group(
    "advocate", "critic", "analyst",
    payload={"topic": "AI regulation"},
    name="research",
    timeout=30.0,
    cancel_on_error=True,  # Cancel all on first error (default)
) as tg:
    # Iterate over intermediate events while waiting
    async for event in tg:
        if event.type == TaskGroupEvent.UPDATE:
            print(f"{event.agent_name}: {event.data}")
        elif event.type == TaskGroupEvent.STREAM_DATA:
            print(f"{event.agent_name} streaming: {event.data}")

# After completion, all responses are available
for agent_name, result in tg.responses.items():
    print(f"{agent_name}: {result}")
```

### Task Group Features

- **Automatic ready-wait** — Waits for agents to register before sending requests
- **Timeout** — Covers both the ready-wait and task execution
- **Cancel on error** — Cancels all workers if any responds with `ERROR` or `FAILED`
- **Async iteration** — Receive updates and streaming data while waiting
- **Partial results** — `tg.responses` is available even after an exception

### Error Handling

```python
from pipecat_subagents.agents.task_context import TaskGroupError

try:
    async with self.task_group("w1", "w2", payload=data, timeout=10.0) as tg:
        async for event in tg:
            pass
except TaskGroupError as e:
    # Timeout, worker error, or ready-wait failure
    print(f"Error: {e}")
    print(f"Partial results: {tg.responses}")
```

## Example: Parallel Debate

A voice agent dispatches research to three workers and synthesizes their findings:

```python
class ModeratorAgent(LLMAgent):
    @tool
    async def research_topic(self, params, topic: str):
        """Gather perspectives from three specialized workers."""
        async with self.task_group(
            "advocate", "critic", "analyst",
            payload={"topic": topic},
            timeout=30.0,
        ) as tg:
            async for event in tg:
                logger.info(f"{event.agent_name}: {event.type}")

        # Synthesize all perspectives
        perspectives = [r["text"] for r in tg.responses.values()]
        summary = f"Topic: {topic}\n\n" + "\n\n".join(perspectives)
        await params.result_callback(summary)
```

## Parent Callbacks

If you prefer callbacks over the context manager:

```python
class MyAgent(BaseAgent):
    async def on_task_response(self, message):
        """Individual worker response."""
        print(f"{message.source}: {message.response}")

    async def on_task_completed(self, result):
        """All workers in the group have responded."""
        for name, response in result.responses.items():
            print(f"{name}: {response}")

    async def on_task_error(self, message):
        """Worker error (when cancel_on_error=True)."""
        print(f"Error from {message.source}: {message.response}")

    async def on_task_update(self, message):
        """Progress update from a worker."""
        print(f"Update from {message.source}: {message.data}")
```
