This guide walks you through building a two-agent system where a greeter agent hands off to a support agent mid-conversation.

## Prerequisites

- Python 3.10+
- `pipecat-ai` installed
- API keys for your chosen STT, LLM, and TTS services

## Install

```bash
pip install pipecat-ai-subagents
```

## Single Agent

The simplest usage is a single agent with a complete voice pipeline:

```python
from pipecat.pipeline import Pipeline
from pipecat_subagents.agents import BaseAgent
from pipecat_subagents.runner import AgentRunner


class MyAgent(BaseAgent):
    async def build_pipeline(self) -> Pipeline:
        # Build your normal Pipecat pipeline here
        return Pipeline([
            transport.input(),
            stt,
            context_aggregator.user(),
            llm,
            tts,
            transport.output(),
            context_aggregator.assistant(),
        ])


async def main():
    runner = AgentRunner()
    agent = MyAgent("my_agent", bus=runner.bus)
    await runner.add_agent(agent)
    await runner.run()
```

## Two-Agent Handoff

For multi-agent systems, the key pattern is:

1. A **main agent** owns the transport and places a `BusBridgeProcessor` in its pipeline
2. **Voice agents** (LLM agents) run conversation logic and transfer between each other

### Main Agent

The main agent handles audio I/O and bridges frames to the bus:

```python
from pipecat.pipeline import Pipeline
from pipecat_subagents.agents import BaseAgent
from pipecat_subagents.bus import BusBridgeProcessor


class MainAgent(BaseAgent):
    async def build_pipeline(self) -> Pipeline:
        bridge = BusBridgeProcessor(
            bus=self.bus,
            agent_name=self.name,
        )

        return Pipeline([
            self.transport.input(),
            stt,
            context_aggregator.user(),
            bridge,          # Routes frames to the active voice agent
            tts,
            self.transport.output(),
            context_aggregator.assistant(),
        ])

    async def on_agent_ready(self, data):
        # Activate the greeter when it registers
        if data.agent_name == "greeter":
            await self.activate_agent("greeter")
```

### Voice Agents

Voice agents use `LLMAgent` with a `@tool` for handoff:

```python
from pipecat_subagents.agents import LLMAgent
from pipecat_subagents.agents.llm_agent import LLMAgentActivationArgs
from pipecat.services.openai.llm import OpenAILLMService


class GreeterAgent(LLMAgent):
    def __init__(self, name, *, bus):
        super().__init__(name, bus=bus, bridged=())

    def build_llm(self):
        return OpenAILLMService(
            api_key=OPENAI_KEY,
            model="gpt-4o",
            system_prompt="You are a friendly greeter. If the user needs "
                "technical help, transfer them to the support agent.",
        )

    @tool
    async def transfer_to_support(self, params, reason: str):
        """Transfer the user to the support team."""
        await self.handoff_to("support", args=LLMAgentActivationArgs(
            messages=[{"role": "user", "content": reason}]
        ))


class SupportAgent(LLMAgent):
    def __init__(self, name, *, bus):
        super().__init__(name, bus=bus, bridged=())

    def build_llm(self):
        return OpenAILLMService(
            api_key=OPENAI_KEY,
            model="gpt-4o",
            system_prompt="You are a technical support specialist.",
        )

    @tool
    async def transfer_to_greeter(self, params, reason: str):
        """Transfer the user back to the greeter."""
        await self.handoff_to("greeter", args=LLMAgentActivationArgs(
            messages=[{"role": "user", "content": reason}]
        ))
```

### Putting It Together

```python
async def main():
    runner = AgentRunner()

    main_agent = MainAgent("main", bus=runner.bus)
    greeter = GreeterAgent("greeter", bus=runner.bus)
    support = SupportAgent("support", bus=runner.bus)

    await runner.add_agent(main_agent)
    await main_agent.add_agent(greeter)
    await main_agent.add_agent(support)

    await runner.run()
```

When the user asks for technical help, the greeter's LLM calls `transfer_to_support`, which deactivates the greeter and activates the support agent — the conversation continues seamlessly.

## Key Concepts

| Concept                  | Description                                                        |
| ------------------------ | ------------------------------------------------------------------ |
| `AgentRunner`            | Entry point that manages agent lifecycles and the shared bus       |
| `BusBridgeProcessor`     | Pipeline processor that routes frames between transport and bus    |
| `bridged=()`             | Makes an agent receive frames from the bus (all bridges)           |
| `handoff_to()`           | Deactivates self and activates another agent                       |
| `LLMAgentActivationArgs` | Injects messages into the target agent's LLM context on activation |
