Pipecat Subagents is a framework for building multi-agent systems where each agent runs its own Pipecat pipeline and communicates with other agents through a shared message bus. Agents can hand off conversations, dispatch tasks in parallel, and stream results — all while maintaining the same programming model whether running locally or distributed across machines.

## Key Capabilities

- **Agent handoff**: Transfer control between agents mid-conversation, seamlessly switching context and capabilities
- **Task coordination**: Dispatch work to specialized workers in parallel, with timeouts, cancellation, and streaming results
- **Per-agent pipelines**: Every agent runs its own Pipecat pipeline with full access to services and processors
- **Mixed agent types**: Combine free-form LLM agents with structured Pipecat Flows agents
- **Distributed**: Run agents across machines via a shared bus that works the same locally or over the network

## Use Cases

- **Customer support**: Specialist agents with dedicated tools and context, transferring seamlessly between departments
- **Parallel research**: Dispatch research to multiple worker agents and synthesize findings
- **Structured workflows**: Route conversations through specialized agents for intake, processing, and resolution
- **Video games**: Multiple LLMs independently controlling characters, environments, or game mechanics

A **main agent** owns the transport (microphone, speakers) and places a bus bridge in its pipeline. **Voice agents** run LLMs and handle conversation logic. **Worker agents** process tasks dispatched by other agents.
