## How It All Fits Together

A typical Pipecat application has three layers:

1. **Server** — A Pipecat pipeline processes audio, runs LLMs, and generates speech in real-time
2. **Client** — A Pipecat Client SDK connects users via browser or mobile app
3. **Infrastructure** — Pipecat Cloud (or self-hosted) handles deployment and scaling

```
┌─────────────┐     WebRTC/WS     ┌──────────────────────────┐
│   Client    │ ◄──────────────► │   Pipecat Pipeline       │
│   (SDK)     │                   │   STT → LLM → TTS        │
└─────────────┘                   └──────────────────────────┘
                                            │
                                  ┌─────────┴────────┐
                                  │  Pipecat Cloud /  │
                                  │  Self-hosted      │
                                  └──────────────────┘
```

For multi-agent systems, Pipecat Subagents adds a coordination layer where multiple pipelines communicate through a shared message bus.
