All examples are available in the [pipecat repository](https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker).

## Local examples

These examples run all agents in a single process using the default `AsyncQueueBus`.

<CardGroup cols={2}>
  <Card
    title="Two LLM Agents"
    icon="arrows-repeat"
    href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/local-handoff/local-handoff-two-agents.py"
  >
    A greeter and a support agent that transfer control between each other. Demonstrates agent handoff with `activate_worker()` and the `@tool` decorator.
  </Card>

<Card
  title="Two LLM Agents with TTS"
  icon="microphone"
  href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/local-handoff/local-handoff-two-agents-tts.py"
>
  Same as above, but each agent has its own TTS with a distinct voice. The main
  agent has no TTS -- audio comes from the active LLM agent through the bus.
</Card>

<Card
  title="Parallel Debate"
  icon="comments"
  href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/parallel-debate/parallel-debate.py"
>
  A moderator dispatches a topic to three agents in parallel using
  `job_group()`. Each one argues from a different perspective. Demonstrates job
  coordination.
</Card>

  <Card
    title="Voice Code Assistant"
    icon="code"
    href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/code-assistant"
  >
    A voice agent backed by a code agent that uses Claude Agent SDK with tools (Read, Bash, Glob, Grep) to answer coding questions. Demonstrates job-based agent integration.
  </Card>
</CardGroup>

## Distributed examples

These examples run agents across separate processes or machines, communicating via Redis, PGMQ, or WebSocket.

<CardGroup cols={2}>
  <Card
    title="Redis Handoff"
    icon="server"
    href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/distributed-handoff/redis-handoff"
  >
    The two-agent handoff split across separate processes using `RedisBus`. The main transport agent runs independently from the LLM agents.
  </Card>

<Card
  title="PGMQ Handoff"
  icon="database"
  href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/distributed-handoff/pgmq-handoff"
>
  The same distributed handoff on a `PgmqBus` (Postgres / Supabase), for
  ops-friendly infrastructure.
</Card>

  <Card
    title="Remote Proxy Assistant"
    icon="globe"
    href="https://github.com/pipecat-ai/pipecat/tree/main/examples/multi-worker/remote-proxy-assistant"
  >
    A main agent connects to a remote LLM server over WebSocket using proxy agents. Demonstrates point-to-point distributed deployment.
  </Card>
</CardGroup>
