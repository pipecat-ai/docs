Pipecat is published on PyPI as [`pipecat-ai`](https://pypi.org/project/pipecat-ai/). Add it to any Python project with [`uv`](https://docs.astral.sh/uv/).

<Note>
  New to Pipecat? The [Quickstart](/pipecat/get-started/quickstart) scaffolds a
  full project for you. This page is the reference for adding Pipecat to an
  existing project or pinning a specific version.
</Note>

## Why uv

Pipecat docs standardize on `uv` for installation and dependency management. `uv` is a fast Python package and project manager from Astral. Compared to `pip`, it:

- Installs and resolves dependencies 10–100x faster
- Manages Python versions for you (no separate `pyenv` setup)
- Generates and respects a lockfile (`uv.lock`) by default for reproducible installs
- Replaces `pip`, `pip-tools`, `virtualenv`, `pipx`, and `poetry` with one tool

The Pipecat CLI scaffolds projects with `uv`, so using `uv` here keeps your local workflow consistent with what `pipecat init` produces.

<Tip>
  If you prefer `pip`, every `uv add` command in these docs has a `pip install`
  equivalent. Just substitute the command — the package names and extras are
  identical.
</Tip>

## Requirements

- Python 3.11 or later (uv can install this for you)
- [`uv`](https://docs.astral.sh/uv/getting-started/installation/) installed

## Install Pipecat

Pipecat ships as a base package with optional **extras** for each transport, AI service, and utility. Install only what you need.

```bash
uv add "pipecat-ai[daily,deepgram,openai,cartesia,silero]"
```

This adds Daily transport, Deepgram STT, OpenAI LLM, Cartesia TTS, and Silero VAD to your project. Swap the extras for your stack.

<Note>
  `uv add` requires a `uv`-managed project (a `pyproject.toml`). If you're
  working in a plain virtual environment, use `uv pip install` instead
  (`uv pip` is a drop-in replacement for `pip`):

```bash
uv pip install "pipecat-ai[daily,deepgram,openai,cartesia,silero]"
```

</Note>

### Available extras

Extras are organized by category:

| Category         | Examples                                                                                   |
| ---------------- | ------------------------------------------------------------------------------------------ |
| Transports       | `daily`, `livekit`, `webrtc`, `websocket`, `lemonslice`                                    |
| STT              | `deepgram`, `assemblyai`, `gladia`, `soniox`, `speechmatics`, `whisper`                    |
| LLM              | `openai`, `anthropic`, `google`, `gemini`, `groq`, `mistral`, `aws`, `azure`, `perplexity` |
| TTS              | `cartesia`, `elevenlabs`, `rime`, `lmnt`, `neuphonic`, `piper`, `kokoro`                   |
| Vision / Video   | `tavus`, `heygen`, `simli`, `moondream`, `fal`                                             |
| Audio processing | `silero`, `koala`, `rnnoise`                                                               |
| Utilities        | `runner`, `mcp`, `tracing`, `sentry`, `langchain`, `mem0`                                  |

Each [service page](/api-reference/server/services/supported-services) lists the exact extra to install.

## Install the Pipecat CLI

The CLI scaffolds projects and deploys to Pipecat Cloud. Install it as a global tool:

```bash
uv tool install pipecat-ai-cli
```

See [Build Your Next Bot](/pipecat/get-started/build-your-next-bot) for the scaffold workflow.

## Install Pipecat Flows (optional)

[Pipecat Flows](/pipecat-flows/introduction) is an optional package for building structured, node-based conversations. You only need it for complex flows where you want deterministic conversation paths (booking flows, intake forms, multi-step processes). For most voice agents, the base `pipecat-ai` package is enough.

If you do need it, install it alongside `pipecat-ai`:

```bash
uv add pipecat-ai-flows
```

## Verify your installation

Check the installed version of any package from inside your project:

```bash
uv pip show pipecat-ai
uv pip show pipecat-ai-flows
pipecat --version
```

Or from Python:

```python
from importlib.metadata import version
print(version("pipecat-ai"))
```

## Pinning a version

For reproducible builds, pin to an exact version:

```bash
uv add "pipecat-ai[daily,openai,cartesia]==1.0.0"
```

Or use a compatible-release specifier to allow patch updates:

```bash
uv add "pipecat-ai[daily,openai,cartesia]~=1.0"
```

`uv` writes the resolved versions to `uv.lock`, so anyone running `uv sync` gets the same install.

## Upgrading

To upgrade Pipecat to the latest version allowed by your version constraints:

```bash
uv lock --upgrade-package pipecat-ai
uv sync
```

To upgrade everything in your project:

```bash
uv sync --upgrade
```

If you're moving from a pre-1.0 release, see the [1.0 migration guide](/pipecat/migration/migration-1.0).
